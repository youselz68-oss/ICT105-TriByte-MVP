# Initial IT Venture Idea Log

## Idea 1: HallPass

### Problem Area
International students do not know where to find honest reviews about dorms and cafeterias. They also find it hard to know basic things like where to eat, what the dorms are like, and how to get around campus when they first arrive.

### Target Users
International students starting university for the first time.

### Current Alternative
Students usually ask friends, search on Google, or look at the university website. However, these sources are not always honest, up to date, or written by real students.

### Proposed IT Solution
A website where students can read and write reviews about dorms and cafeterias. The website also has simple guides to help new students settle in, such as where to eat, what to bring, and how to find help on campus.

### Possible Technology
- Web application
- Cloud-based system to store reviews and user data
- User authentication system
- Review and rating features

### Why This Is Suitable
International students need a simple and reliable source of information when they first arrive. A website is easy to access from any device without needing to download anything, making it ideal for students who are new to campus.

## Idea 2: StudyBuddy

### Problem Area
Many students struggle to find study partners, group members, or classmates taking the same subjects.

### Target Users
University students.

### Current Alternative
Students usually ask in class, use social media groups, or message friends to find study partners.

### Proposed IT Solution
A web platform where students can create profiles, list the subjects they are taking, and connect with other students looking for study partners or project teammates.

### Possible Technology
- Web application
- Student profile management
- Search and matching system
- Cloud database
- Messaging feature

### Why This Is Suitable
The project involves user accounts, databases, and search functionality, making it suitable for learning web development concepts while solving a common student problem.

## Idea 3: Campus Event Hub

### Problem Area
Students often miss campus events because information is spread across different platforms and notices.

### Target Users
University students, student clubs, and university staff.

### Current Alternative
Students rely on posters, social media pages, group chats, or word of mouth to learn about events.

### Proposed IT Solution
A centralized website where clubs and organizations can post upcoming events, and students can browse, search, and register for activities happening on campus.

### Possible Technology
- Web application
- Event management system
- Cloud database
- User registration and login
- Event calendar feature

### Why This Is Suitable
The project can be developed as a prototype using web technologies and database concepts. It includes practical features such as event posting, searching, and registration while remaining achievable within a semester.
